# Maintenance Log: 2026-07-22

## Summary of Operations
Created and configured a new ZFS dataset (`local-fast-zfs/kevin`) on the Proxmox VE host `misty` to be exported as a read-write NFS share called `kevin`. The share is configured with a 500 GB storage quota and is accessible to both the `fog` (`10.7.82.0/24`) and `lodge` (`10.5.110.0/24`) subnets.

---

## 1. ZFS Dataset Creation & Quota Configuration
*   **Action:** Provisioned a ZFS dataset under the fast NVMe pool.
*   **Command:** `zfs create local-fast-zfs/kevin`
*   **Storage Quota:** Applied a hard limit of **500 GB** to prevent excessive space consumption.
*   **Command:** `zfs set quota=500G local-fast-zfs/kevin`

---

## 2. NFS Share Configuration & Export
*   **Action:** Shared the dataset via NFS, authorizing access from both local subnets.
*   **Command:** `zfs set sharenfs="rw=@10.7.82.0/24:@10.5.110.0/24,all_squash,anonuid=1000,anongid=1000,async" local-fast-zfs/kevin`
*   **Permissions Mapping:** Configured `all_squash` to map all client connections to UID `1000` / GID `1000` (`nick:nick`) for clean permission ownership.
*   **Reload Exports:** Executed `exportfs -ra` on the host to update active exports.

---

## 3. Directory Permissions Setup
*   **Action:** Set directory ownership and broad permissions on the host mountpoint.
*   **Commands:**
    ```bash
    chown -R 1000:1000 /local-fast-zfs/kevin
    chmod 777 /local-fast-zfs/kevin
    ```
